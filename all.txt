Scheduling Build Jobs using Jenkins 
AIM 
To schedule Jenkins build jobs using Poll SCM and Build periodically triggers. 
THEORY 
Scheduling in Jenkins: 
Scheduling is the process of executing Jenkins jobs automatically without manual 
intervention. Jenkins provides different triggers to run jobs based on time or code 
changes. 
Poll SCM: 
Poll SCM is a build trigger in Jenkins that periodically checks the source code repository 
for any changes. If changes are detected, Jenkins automatically triggers the build. 
Build Periodically: 
Build periodically is a trigger in Jenkins that executes the job automatically at specified 
time intervals, regardless of any changes in the repository. 
CRON Expression: 
CRON expression is a time-based scheduling format used in Jenkins to define when a 
job should run. 
Example: 
H/5 * * * * 
This means the job runs every 5 minutes.

Maven POM, Lifecycle, Dependencies, Plugins & Repositories 
 
  AIM 
To study Maven POM (pom.xml), build lifecycle, repositories, dependencies, plugins, 
and related concepts. 
 
  THEORY 
 
What is maven? 
     Maven is a build automation tool used for Java projects. 
     It automates tasks like compiling, testing, and packaging code. 
     It also manages project dependencies using the pom.xml file. 
   1. Maven POM (pom.xml) 
     POM stands for Project Object Model 
     It is the main configuration file in a Maven project. 
It defines: 
• Project metadata (GroupId, ArtifactId, Version) 
• Dependencies (libraries required) 
• Plugins 
• Build configuration 
 
   2. GroupId, ArtifactId, Version 
• GroupId → Identifies organization/project namespace 
     Example: com.example 
• ArtifactId → Name of the project 
     Example: myapp 
• Version 
o 1.0-SNAPSHOT → Development version 
o 1.0 → Stable version 
 
   3. Snapshot 
     Snapshot is a development version 
     It changes frequently during development 
 
   4. Maven Dependencies 
     Dependencies are external libraries required by the project. 
Example: 
<dependencies> 
  <dependency> 
    <groupId>org.junit.jupiter</groupId> 
    <artifactId>junit-jupiter</artifactId> 
    <version>5.9.3</version> 
    <scope>test</scope> 
  </dependency> 
</dependencies> 
     Used for testing purposes 
 
   5. Maven Plugins 
     Plugins extend Maven functionality. 
Common plugins: 
• Compiler Plugin → compiles Java code 
• Surefire Plugin → runs test cases 
 
Example: 
<build> 
  <plugins> 
    <plugin> 
      <groupId>org.apache.maven.plugins</groupId> 
      <artifactId>maven-compiler-plugin</artifactId> 
      <version>3.11.0</version> 
      <configuration> 
        <source>17</source> 
        <target>17</target> 
      </configuration> 
    </plugin> 
  </plugins> 
</build> 
 
   6. Maven Build Lifecycle 
Maven consists of three lifecycles: 
 
  1. Clean Lifecycle 
mvn clean 
     Deletes target/ directory 
 
  2. Default Lifecycle 
Phase Command Purpose 
Validate mvn validate Check project 
Compile mvn compile Compile code 
Test mvn test Run tests 
Package mvn package Create JAR 
Install mvn install Store in local repo 
 
  3. Site Lifecycle 
mvn site 
     Generates project documentation 
 
   7. Maven Repositories 
 
  Local Repository 
     Stored on local system 
   Location: 
~/.m2/repository 
     Stores downloaded dependencies 
 
  Global (Central) Repository 
     Online repository 
     Example: 
https://repo.maven.apache.org/maven2/ 
     Contains libraries, frameworks, plugins 
 
    PROCEDURE (EXECUTION PART)

sudo apt update 
 
  Step 2: Install Java (required) 
Check:

  Step 4: Verify Maven 
mvn -version 
     Shows Maven + Java version   
 
  Step 5: Create Maven Project

  Step 6: Go to project folder 
cd myapp

  Step 9: Package project

Ansible Playbooks, Variables, Loops, Handlers, Templates & Roles 
AIM 
To install Ansible and execute playbooks using variables, loops, conditionals, handlers, 
templates, and roles. 
REQUIREMENTS 
• Ubuntu OS  
• Internet connection  
• Sudo privileges  
THEORY (WRITE THIS) 
Ansible 
An automation tool used for configuration management and deployment. 
Playbook 
A YAML file used to automate tasks. 
YAML 
Human-readable configuration language. 
Modules 
Predefined functions (apt, copy, service). 
Variables 
Store reusable values. 
Handlers 
Triggered tasks (run when notified). 
Loops 
Repeat tasks multiple times. 
Conditionals 
Execute tasks based on conditions. 
Templates 
Dynamic configuration files. 
Roles 
Reusable structured components. 
PROCEDURE (STEP-BY-STEP EXECUTION) 
STEP 1: Install Ansible 
sudo apt update 
sudo apt install ansible -y 
ansible --version 
STEP 2: Create Project Directory 
mkdir ~/ansible 
cd ~/ansible 
STEP 3: Create Inventory File 
nano hosts.ini 
[webservers] 
localhost ansible_connection=local 
STEP 4: Create Basic Playbook 
nano local_playbook.yml --- - name: Install Apache on Local Machine 
hosts: webservers 
become: yes 
  tasks: 
    - name: Install Apache 
      apt: 
        name: apache2 
        state: present 
 
    - name: Start Apache 
      service: 
        name: apache2 
        state: started 
        enabled: yes 
 
    - name: Create index.html 
      copy: 
        dest: /var/www/html/index.html 
        content: "<h1>Hello from Apache!</h1>" 
 
  STEP 5: Run Playbook 
ansible-playbook -i hosts.ini local_playbook.yml --ask-become-pass 
     Enter system password 
 
  STEP 6: Verify Output 
systemctl status apache2 
Open browser: 
http://localhost 
     Output: Hello from Apache! 
 
  STEP 7: Java Program Deployment 
nano HelloWorld.java 
public class HelloWorld { 
    public static void main(String[] args) { 
        System.out.println("Hello from Java!"); 
    } 
} 
Compile & run: 
javac HelloWorld.java 
java HelloWorld 
 
Display in browser: 
java HelloWorld | sudo tee /var/www/html/java_output.html 
Open: 
http://localhost/java_output.html 
 
  STEP 8: Advanced Playbook (Handlers + Variables) 
Create file: 
mkdir -p files 
nano files/index.html 
<h1>Hello from Ansible!</h1> 
 
Create playbook: 
nano site.yml - name: Configure web servers 
  hosts: webservers 
  become: yes 
 
  vars: 
    app_dir: /var/www/html 
 
  tasks: 
    - name: Install Apache 
      apt: 
        name: apache2 
        state: present 
 
    - name: Deploy index.html 
      copy: 
        src: index.html 
        dest: "{{ app_dir }}/index.html" 
      notify: Restart Apache 
 
  handlers: 
    - name: Restart Apache 
      service: 
        name: apache2 
        state: restarted 
 
Run: 
ansible-playbook -i hosts.ini site.yml --ask-become-pass 
 
  STEP 9: Loops - name: Install packages 
  apt: 
    name: "{{ item }}" 
    state: present 
  loop: 
    - git 
    - curl 
 
  STEP 10: Conditionals 
when: ansible_os_family == "Debian" 
 
  STEP 11: Templates 
mkdir templates 
nano templates/config.j2 
ServerName {{ inventory_hostname }} 
 
  STEP 12: Roles 
ansible-galaxy init webserver
